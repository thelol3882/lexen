# Lexen — личная стратегия (не фичи)

## Фаза A — стабилизация внутри fcaibyn (сейчас, 1–2 дня)

- Агенты проходят по `web/app/` по правилам из `scripts/lexen/RULES.md`
- Цель: `pnpm lexen check` и `pnpm lint` — зелёные
- Commit → merge в main

**Критерий завершения:** ноль нарушений правил, ноль drift, ноль `MISSING_MESSAGE` в runtime

---

## Фаза B — вынос в отдельный репо (когда стартуешь RN приложение)

1. Создать `github.com/thelol3882/lexen` (public repo)
2. Скопировать `scripts/lexen/` туда, запушить
3. В fcaibyn: `pnpm add github:thelol3882/lexen`, удалить `scripts/lexen/`
4. В RN проекте: `pnpm add github:thelol3882/lexen` + `lexen init --preset=react-i18next`

**Смысл:** проверить переносимость на реальном втором проекте. То что работает в 1 проекте ≠ то что работает в 2-х.

**Критерий:** оба проекта используют lexen без форков, без локальных патчей

---

## Фаза C — официальный релиз на npm (необязательная, только после успешной B)

Pre-publication hardening (~15 минут):

- Path traversal защита в `util/paths.ts`
- Prototype pollution safety в JSON merge (`locales.ts`)
- `.npmignore` / `"files"` в `package.json` — не шипать `__tests__/fixtures/`
- `LICENSE` файл (MIT)
- README в правильном тоне (см. раздел "Mindset" ниже)
- `npm publish`
- Минимальный CI (fixture tests на Node 20 + 22)

---

## Триггеры перехода между фазами

| Переход | Триггер |
|---------|---------|
| A → B   | Когда начинаешь RN проект **ИЛИ** когда ловишь мысль "ох, копипастить lexen в новый проект — лень" |
| B → C   | Когда lexen гладко работает в 2+ реальных твоих проектах 3+ месяца без форков/патчей |

---

## Когда НЕ публиковать вообще

- Фаза A раскрыла что lexen слишком завязан на fcaibyn-specific вещи (path conventions, feature/widget assumptions)
- Breaking changes идут часто (раз в неделю и чаще) — рано для публичного API
- Реально нет времени даже на "посмотреть issue раз в месяц" — тогда держи в своём монорепо

---

## Mindset

- **Никаких** "actively maintained", "community-driven" в README
- Обновляешь когда тебе нужна фича, не когда просят
- Issues смотришь когда настроение, не по расписанию
- Нравится чей-то PR → мержишь; не нравится / узкий кейс → вежливый "not for me, feel free to fork"
- Нет моральных обязательств перед пользователями публичного репо — ты им ничего не должен

> Популярность lexen — это бонус, а не цель. Цель — тебе удобно работать с локалями в твоих проектах.
> Если кому-то ещё поможет — классно, но не мотивация.
